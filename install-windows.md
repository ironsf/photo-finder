# Установка и запуск на Windows 11

Инструкция предполагает, что **Python уже установлен** (3.10 или новее) и добавлен в PATH.
Проверить можно командой в «Командной строке» (cmd) или PowerShell:

```
python --version
```

Если выводит `Python 3.1x.x` — всё в порядке. Если команда не распознаётся, попробуйте `py --version`
и далее используйте `py` вместо `python` во всех командах ниже.

> Tkinter (окно проверки) на Windows идёт в комплекте с официальным Python — отдельно ставить не нужно.
> В отличие от macOS, никаких дополнительных пакетов для интерфейса не требуется.

---

## Шаг 1. Скопировать проект

Скопируйте всю папку `photoFinder` на компьютер, например в `C:\photoFinder`.

⚠️ **Важно:** проверьте, что вместе с проектом скопировался файл **`.env`** (в нём ключи API).
Он скрытый — включите в Проводнике «Вид → Показать → Скрытые элементы», чтобы его увидеть.
Если `.env` не скопировался, создайте его вручную (см. Шаг 4).

---

## Шаг 2. Открыть терминал в папке проекта

Откройте папку `photoFinder` в Проводнике, щёлкните по адресной строке, введите `cmd` и нажмите Enter —
откроется командная строка прямо в этой папке.

Все команды ниже выполняются в этом окне.

---

## Шаг 3. (Рекомендуется) Виртуальное окружение

Чтобы не засорять системный Python, создайте отдельное окружение для проекта:

```
python -m venv .venv
.venv\Scripts\activate
```

После активации в начале строки появится `(.venv)`. Этот шаг необязателен, но желателен.
Если решили без него — просто пропустите и ставьте пакеты в системный Python.

---

## Шаг 4. Установить зависимости

```
pip install openpyxl Pillow transformers
pip install torch --index-url https://download.pytorch.org/whl/cpu
```

**Почему torch отдельной командой:** если поставить его обычным `pip install torch`, pip может скачать
огромную версию под видеокарту (CUDA, несколько гигабайт). Модель распознавания у нас работает на процессоре,
поэтому ставим лёгкую CPU-версию через специальный индекс.

Установка torch занимает пару минут — это нормально.

---

## Шаг 5. Ключи API (файл `.env`)

В папке проекта должен быть файл **`.env`** такого вида:

```
BRAVE_API_KEY=ваш_ключ_brave
SERPAPI_KEY=ваш_ключ_serpapi
```

Если файла нет — скопируйте `.env.example`, переименуйте копию в `.env` и впишите ключи после `=`
(без кавычек, без пробелов). Ключи в код не попадают и в репозиторий не коммитятся.

---

## Шаг 6. Положить файл с товарами

Скопируйте `.xlsx` с товарами прямо в папку `photoFinder`:
- столбец **A** — штрихкод,
- столбец **B** — название,
- строки заголовка нет (данные с первой строки).

Программа сама возьмёт первый `.xlsx`, который найдёт в своей папке.

---

## Шаг 7. Запуск

```
python main.py
```

**При первом запуске** скачается модель проверки соответствия картинок (SigLIP2, ~1.4 ГБ) с Hugging Face —
нужен интернет, займёт несколько минут. Это разово: при последующих запусках модель берётся из кэша.

После загрузки откроется окно проверки.

---

## Как пользоваться окном

Программа показывает картинку + название товара. Управление — кнопками или горячими клавишами:

| Клавиша | Действие |
|---|---|
| **Y** или **Enter** | Да — сохранить эту картинку, перейти к следующему товару |
| **N** или **→** | Нет — показать следующий вариант |
| **B** или **←** | Назад — к предыдущему варианту |
| **S** | Пропустить товар (не сохранять) |
| **Esc** | Выход |

- При высокой уверенности модели картинка **принимается автоматически** через несколько секунд
  (идёт обратный отсчёт) — если не хотите, нажмите любую клавишу, отсчёт отменится.
- Когда варианты в текущем источнике заканчиваются, программа сама подгружает следующий (жать «нет» можно долго).
- **Штрихкод и название можно выделить мышью и скопировать** (Ctrl+C). **Клик по штрихкоду** копирует его
  целиком в буфер обмена — появится надпись «✓ Скопировано в буфер обмена».
- Результат — в папке `images\<штрихкод>.jpg`, прогресс — в `report.csv`.
- Окно можно закрыть в любой момент: при следующем запуске программа продолжит с того же места
  (уже обработанные товары пропускаются).

## Сброс прогресса

Чтобы начать проверку заново (все товары снова появятся):

```
python reset.py          
```

Спросит подтверждение, затем удалит `report.csv` и картинки из `images\`.
Кэш поиска и счётчик расходов при этом **не трогаются** — повторный прогон не будет заново тратить бюджет.

Дополнительно:
```
python reset.py --yes    # без подтверждения
python reset.py --all    # ещё и очистить кэш поиска (следующий прогон платный заново)
```

## Полезные настройки (файл `config.py`)

Откройте `config.py` любым текстовым редактором (например, «Блокнотом») — там наверху понятные параметры:

- `SEARCH_PROVIDERS = ["google", "brave"]` — **источники поиска и их порядок**. Список сверху вниз =
  приоритет. Чтобы выключить источник — уберите его из списка; чтобы поменять приоритет — переставьте.
  Доступно: `"google"` (SerpAPI, качественнее, ~$0.025/запрос), `"brave"` (дешевле, ~$0.005),
  `"yandex"` (Яндекс через SerpAPI), `"bing"` (бесплатно через Playwright, но нестабильно).
- `QUERY_ORDER = ["barcode", "name_barcode", "name"]` — **что искать**. По умолчанию сперва по штрихкоду
  (как ищет оператор вручную), потом «название штрихкод», потом название. Можно оставить только
  `["barcode"]`, если хотите искать строго по штрихкоду.
- `TRANSLATE_TO_ENGLISH = False` — перевод названий на английский для поиска. Чтобы использовать: поставьте
  `True`, добавьте `"name_en"` в `QUERY_ORDER`, и установите пакет: `pip install deep-translator`.
- `MAX_PAID_SPEND_USD = 20.0` — потолок платного расхода. **Перед боевым прогоном на большом списке
  поднимите это значение**, иначе программа остановится по достижении лимита.
  ⚠️ С Google первым каждый товар стоит ~$0.025 (SerpAPI). Хотите дешевле — поставьте `"brave"` первым.
- `MIN_IMAGE_DIMENSION = 650` — минимальный размер «крупной» картинки. Мельче — уходит в конец списка
  и не апскейлится. Можно уменьшить (например, 500), если крупных фото у товаров мало.
- `AUTO_ACCEPT_THRESHOLD = 0.9` — порог автоприёма (0.9 = 90% уверенности). Выше = строже.
- `AUTO_ACCEPT_DELAY_SECONDS = 4` — сколько секунд отсчёт до автоприёма.

После изменения `config.py` сохраните файл и перезапустите `python main.py`.

## Если что-то не работает

- **`'python' не является... командой`** — Python не в PATH. Попробуйте `py main.py` вместо `python main.py`.
- **`ModuleNotFoundError: No module named 'PIL'` (или другого пакета)** — не установлены зависимости
  (Шаг 4), либо вы в другом окружении (если использовали venv — активируйте его: `.venv\Scripts\activate`).
- **Долгая пауза при первом запуске** — это скачивается модель (~1.4 ГБ). Дождитесь, это один раз.
- **`Достигнут лимит платного бюджета`** — поднимите `MAX_PAID_SPEND_USD` в `config.py`.
- **Окно не открывается / ошибка Tkinter** — переустановите Python с python.org (официальный установщик
  включает Tkinter). Проверить: `python -m tkinter` должно открыть маленькое тестовое окно.
